I did this to help me understand whats going on with keyframe animations. This image is 2000px wide and has ten different "frames" of the animation repeated all the way across. The code takes ten "steps" down the image, doing the math for me, literally shifting the image position in the background. So, "play .8s steps(10) infinite"... if you were to change the number of steps to "steps(11)" (like is shown in the animation below) then the animation would change. There are only ten frames on a fixed dimension image, so telling the computer that there are 11 frams then divides the pasted code into eleven parts and essentially renders the animation worthless. I copy and pasted this code, and then destroyed it, for science.
